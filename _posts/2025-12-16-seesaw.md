> 프리미엄 패션 브랜드 이커머스 플랫폼 (1인 풀스택)
{: .prompt-tip }

**개발 기간** 2025.12.02 ~ 2025.12.16 (15일, 매일 평균 4~5시간)
**역할** 1인 풀스택 개발

## 기술 스택

| 분류 | 사용 기술 |
|------|-----------|
| Frontend | Next.js 16 · React 19 · TypeScript · Tailwind CSS 4 · Framer Motion · TanStack Query |
| Backend & DB | Supabase · PostgreSQL · Supabase Auth · Stripe · Resend |
| Deploy | Vercel · Vercel Analytics |

핵심 구현: Supabase RLS 기반 인증/권한, Stripe Webhook 결제 처리, TanStack Query 서버 상태 최적화, Resend 트랜잭션 이메일 자동화

## 주요 기능

- **쇼핑** — 제품 목록/상세/검색/필터, 장바구니 & 위시리스트(DB 연동), 최근 본 상품
- **결제** — Stripe Checkout, 쿠폰/할인 코드, 주문 확인 이메일(Resend)
- **인증** — 로그인/회원가입, 비밀번호 찾기/재설정, 프로필 관리 & 계정 삭제
- **관리자** — 매출/주문 통계 대시보드, 주문·상품·재고·쿠폰 관리

## 트러블슈팅

### 1. 바이브 코딩으로 유려한 디자인 UX 구현

혼자서 디자인부터 개발까지 담당하면서 프리미엄 브랜드에 맞는 고품질 UI를 효율적으로 구현해야 했습니다.

- Cursor AI · v0로 컴포넌트 빠르게 생성, 자연어 반복 프롬프팅
- 최종 디테일은 직접 조정 → Framer Motion 애니메이션, Quick View 모달, 호버 인터랙션
- **디자인 작업 시간 70% 단축**

### 2. Stripe 결제 상태 동기화

결제 후 브라우저 종료/네트워크 문제로 "결제는 됐는데 주문이 없는" 상황이 발생할 수 있었습니다.

- Stripe Webhook으로 `checkout.session.completed` 이벤트 수신
- 서버 사이드에서 주문 생성/상태 업데이트, `metadata`에 주문 정보 저장 → **누락 주문 0건**

### 3. 동적 쿠폰 할인 적용

자체 쿠폰 테이블과 Stripe Coupon 동기화가 필요했습니다.

- 결제 API에서 쿠폰 유효성 검증 후 Stripe Coupon **동적 생성**
- `max_redemptions: 1` 일회용 처리, Checkout Session `discounts`에 적용

## 배운 점

- 결제 같은 중요 로직은 클라이언트가 아닌 서버 사이드에서 처리해야 안정적
- 외부 서비스(Stripe, Supabase) 연동 시 API를 창의적으로 활용하는 법
- AI와 협업하는 개발 방식의 효율성과 가능성
