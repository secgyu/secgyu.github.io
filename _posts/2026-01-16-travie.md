> AI 기반 여행 플래너
{: .prompt-tip }

**개발 기간** 2025.11.07 ~ 2026.01.16 (약 2달, 매일 평균 2시간)
**역할** 프론트엔드 개발

## 프로젝트 개요

여행 계획은 정보 과잉, 동선 최적화, 예산 관리, 결정 장애 등으로 어렵습니다. 트래비는 **"3박4일 도쿄 여행 일정 만들어줘"** 한마디로 완벽한 여행 계획을 완성합니다.

- **AI 일정 생성** — GPT-4o-mini가 맞춤형 일정 자동 생성
- **지도 시각화** — Google Maps로 동선 바로 확인
- **대화형 수정** — 자연스러운 대화로 일정 수정

## 트러블슈팅

### 1. AI 응답을 구조화된 데이터로 파싱

GPT 응답은 자유형식 텍스트라 지도 표시·수정·저장에 필요한 구조화 데이터로 변환하기 까다로웠습니다. JSON 강제는 대화형 UX를 깨고, Function Calling은 스트리밍과 호환이 어려웠습니다.

- 시스템 프롬프트에 출력 형식을 예시와 함께 명시
- `**N일차**` 패턴, `오전/오후 HH:MM - 장소명` 패턴으로 정규식 추출
- 파싱 실패 시 기본 일정 반환 폴백 → **95%+ 파싱 성공률**

### 2. 장소명 → GPS 좌표 변환 (스마트 지오코딩)

한글 장소명을 Google Geocoding에 그대로 넣으면 엉뚱한 위치가 나왔습니다.

- 영문명 + 도시명 조합으로 Geocoding 호출
- 실패 시 OpenAI 좌표 질의(temp 0) → 그래도 실패 시 도시 중심 좌표
- Map 자료구조 캐싱 → **API 호출 비용 70% 절감**

### 3. Google Maps 커스텀 마커

기본 핀으로는 일정 순서를 표현할 수 없었습니다.

- `AdvancedMarkerElement` + DOM(`HTMLDivElement`)으로 숫자 마커 동적 생성
- 선택 여부에 따라 크기/배경색/테두리 변경, `zIndex: 1000`으로 최상단 표시

### 4. 클라이언트 사이드 PDF 생성

Vercel 서버리스(함수 50MB 제한)에서 Puppeteer가 어렵고, jsPDF 직접 출력은 한글이 깨졌습니다.

- 숨겨진 iframe(`-9999px`)에 스타일 적용 HTML 렌더링
- `html2canvas`(scale 2)로 캡처 후 `jsPDF`로 A4 자동 분할 → **서버 비용 0원**, 한글·이모지 정상 출력
