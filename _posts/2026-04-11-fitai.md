> GPT-4o-mini 기반 AI 피트니스 플랫폼
{: .prompt-tip }

**개발 기간** 2026.04.07 ~ 2026.04.11
**역할** 풀스택 개발 (모바일 앱 + 웹 어드민)

## 프로젝트 개요

### 기획 배경

운동 초보자가 어떤 운동을 해야 할지 모르는 문제를 해결하고자 했습니다. 기존 앱은 단순 기록 위주이거나 전문가 수준의 지식을 요구해 진입 장벽이 높았습니다.

### 솔루션

- GPT-4o-mini가 사용자 프로필 기반 맞춤 루틴 자동 생성
- 운동 기록 및 분석 대시보드 제공
- 소셜 커뮤니티를 통한 동기부여

## 기술 스택

| 분류 | 사용 기술 |
|------|-----------|
| Mobile App | React Native 0.81 · Expo 54 · Expo Router 6 · React 19 · TypeScript |
| Web Admin | Next.js 16 · React 19 · Tailwind CSS 4 · App Router |
| Backend & AI | Firebase (Auth · Firestore · Storage) · OpenAI GPT-4o-mini · ExerciseDB API |
| Infra | Firebase Hosting · Vercel |

## 주요 기능

- **AI 운동 루틴 생성** — 프로필(목표/체력/환경/장비) 기반 4~6종목 + 웜업/쿨다운, JSON Schema 구조화 응답, 원클릭 재생성
- **운동 세션 기록** — 세트/무게/횟수 입력, 휴식 타이머, 세션 완료 통계
- **운동 라이브러리** — ExerciseDB 1300+ 운동, 부위/장비/타겟 필터, GIF 미리보기
- **기록 & 분석** — 캘린더 기록, 주간 운동량 차트, 진행 추이 라인 차트, 무드 트래킹
- **소셜 커뮤니티** — 게시글 CRUD, 댓글/좋아요/북마크, 팔로우, 루틴 공유, AI 채팅 어시스턴트
- **웹 어드민** — 유저 관리(검색/필터/CSV), 콘텐츠·신고 관리, 14일 트렌드 분석

## 트러블슈팅

### 1. Firebase Auth 상태 기반 스마트 리다이렉트

인증 상태/온보딩 완료 여부에 따른 3가지 분기 라우팅이 필요했는데, Auth 초기화와 Firestore 조회의 비동기 타이밍 이슈로 화면 깜빡임과 잘못된 리다이렉트가 발생했습니다.

- `onAuthStateChanged` 콜백에서 Firestore 프로필 동시 조회
- `authReady` + `onboardingDone` 2중 상태로 타이밍 제어
- 온보딩 미완료 유저는 `signOut` 후 welcome으로 리다이렉트

### 2. OpenAI 응답 안정성 + 프롬프트 엔지니어링

루틴 JSON 응답이 간헐적으로 스키마와 불일치하고, 한/영 혼재 출력으로 UI 오류가 났습니다.

- `response_format: json_object`로 JSON 응답 강제
- 프롬프트에 정확한 JSON 스키마 명시 + 프로필 영문 매핑
- `temperature: 0.8`로 변형과 일관성 균형

### 3. ExerciseDB API 성능 최적화

1300+ 데이터로 응답이 느리고, AI 생성 운동명과 DB 이름이 불일치했습니다.

- Map 기반 메모리 캐시 (TTL 30분, 키: URL+파라미터)
- 10건 단위 페이지네이션으로 점진적 로딩
- `generateSearchVariants`로 검색어 자동 변형, 이미지 URL은 ID 기반 직접 생성

## 배운 점

- Firebase Auth + Firestore 기반 인증/온보딩 플로우 설계
- OpenAI JSON Schema로 LLM 출력 안정화
- 크로스 플랫폼(모바일+웹) 통합 Firebase 백엔드 경험
